# Ambxst / axctl Ignore Alpha Patch (Temporary) - Handoff

Date: 2026-05-05

## What was broken
- Hyprland config errors were caused by invalid `layerrule` syntax generated by Ambxst/axctl:
  - `ignore_alpha = on`
  - `ignore_alpha = 0.40` (with `=`)

## Root cause found
- `CompositorTomlWriter.qml` (Ambxst shell) writes TOML, then axctl generates Hyprland lines.
- In axctl generator (`pkg/ipc/hyprland/generator.go`), `ignore_alpha` output format was incorrect for Hyprland.

## Temporary fix implemented
- Added a local patched Ambxst package in this repo:
  - `NixOS/pkgs/ambxst-patched/default.nix`
  - `NixOS/pkgs/ambxst-patched/hyprland-layer-ignore-alpha.patch`
- Exposed package in:
  - `NixOS/flake.nix` as `packages.<system>.ambxst-patched`
- Installed with priority 1:
  - `nix profile add --priority 1 .#ambxst-patched`

## Patch behavior
- Removed `ignore_alpha = on` generation.
- Changed `ignore_alpha = %.2f` to `ignore_alpha %.2f`.

## Verification done (before reboot)
- Active `axctl` binary resolved to patched build:
  - `/nix/store/...-axctl-0.0.11-ambxst-ignore-alpha/bin/axctl`
- Regenerated Hyprland config now contains valid entries:
  - `ignore_alpha 1.00`
  - `ignore_alpha 0.40`
  - `ignore_alpha 0.50`
- `hyprctl configerrors` returned empty output (no errors).

## Notes about axctl commands
- `axctl config batch '{"gaps.inner": 8, "gaps.outer": 12}'` -> `Success`
- `axctl config reload` -> `Success`
- `axctl config set gaps.inner 9` -> `Success`
- `axctl config get gaps.inner` still fails in this axctl version (known behavior)
- `axctl config get general:gaps_in` works for readback

## After reboot: quick retest checklist
1. Launch Ambxst if not already running: `ambxst`
2. Confirm generated config lines:
   - `rg "ignore_alpha|layerrule" ~/.local/share/ambxst/hyprland.conf`
3. Confirm Hyprland has no parser errors:
   - `hyprctl configerrors`
4. Optional command test:
   - `axctl config batch '{"gaps.inner": 8, "gaps.outer": 12}'`
   - `axctl config reload`
